
# 하청 공급망 네트워크 분석 프로젝트

## 1. 프로젝트 소개

### 1. 프로젝트 소개 및 배경

#### 하청 공급망 차수 식별의 중요성

- 한국 경제는 대기업과 하청기업 간의 뚜렷한 위계적 구조가 특징
	- 소수의 대기업이 다수의 중소기업과 거래하는 수직적 구조 고착화
	- 대기업-중소기업 간 임금, 생산성, 수익성 격차 지속 확대
	- 이러한 구조적 특성이 한국 경제의 양극화에 기여한다는 지적
- 이러한 위계 구조가 기업의 임금, 고용 등 주요 의사결정에 미치는 영향을 분석하기 위해서는 공급망 내 기업의 정확한 차수 식별이 필수(Hong and Jung, 2015)
	- 단순한 규모나 산업 분류로는 기업 간 관계의 복잡성을 포착하기 어려움
	- 공급망 내 위치에 따라 협상력과 거래조건이 달라질 수 있음
	- 정확한 차수 식별은 정책 설계의 기초자료로도 활용 가능
- 최근의 실증 연구들은 우리의 직관과 실제 데이터 간에 상당한 괴리가 있음을 보여줌 
	- 도요타 공급망 분석에서 피라미드형이 아닌 항아리형 구조 발견(Nakano and White, 2007)
	- 단순한 수직적 구조가 아닌 복잡한 네트워크 구조의 존재 확인
	- 직관에 기반한 정책의 한계와 데이터 기반 분석의 필요성 제기

### 네트워크 분석 방법론의 적용

- 본 프로젝트는 소셜 네트워크 분석(SNA) 방법론을 활용하여 기업 거래 네트워크의 구조적 특성을 파악하고자 함
- 소셜 네트워크 분석(SNA)은 다음과 같은 이유로 기업 간 거래관계 연구에 특히 적합

#### 이론적 적합성
- 네트워크 내 권력과 영향력 구조 파악이 주요 목적
- 행위자들 간의 관계적 속성에 초점
- 구조적 위치가 행위자의 기회와 제약을 결정한다는 이론적 전제
	- e.g. 네트워크 내 기업의 위치는 가격 결정과 거래 조건에 중요한 영향 (Kranton and Minehart ,2001))
- 위계적 구조의 식별과 분석에 특화된 방법론 보유
#### 실증적 적합성
- 대규모 관계형 데이터의 체계적 분석 가능
- 직접적 관계뿐 아니라 간접적 관계의 영향도 포착
- 위치의 측정과 비교를 위한 계량적 지표 제공
- 시각화를 통한 직관적 이해 지원

#### 초기 접근: 단순 중심성 개념
- 연결정도(degree) 중심성 위주의 분석
    - 직접적인 거래 관계의 수 측정
    - 계산이 단순하고 직관적 해석 용이
    - 그러나 실제 권력구조와 불일치 발견
    - 네트워크의 전체 구조 반영 못함
#### 구조적 접근으로의 확장
- 매개(betweenness) 중심성 도입
    - 정보/자원 흐름의 통제력 측정
    - 구조적 공백의 이점 포착
    - 간접적 영향력 고려 가능
    - Freeman(1979)의 이론적 기여
- 고유벡터(eigenvector) 중심성
    - 연결된 노드의 중요도 반영
    - 재귀적 권력 개념의 도입
    - Bonacich(1987)의 방법론적 혁신
    - 그러나 방향성 고려 못함
#### 방향성
- 특권성(prestige) 지수
    - 입력/출력 연결의 구분
    - 위계적 구조의 명시적 고려
    - 권력의 비대칭성 포착
    - 산업 구조의 현실에 부합
- PageRank 알고리즘
    - 구글의 웹페이지 순위 알고리즘 응용
    - 방향성 네트워크의 위계 구조 포착
    - 전역적(global) 중요도 측정
    - 계산의 효율성과 확장성 확보

## 2. 방법론

### 2.1 네트워크 분석 기초

본 프로젝트에서는 다양한 네트워크 중심성 지표를 활용했습니다:

1. **기본 중심성 지표**
   - Degree Centrality: 직접적 거래 관계의 수
   - Betweenness Centrality: 기업 간 거래 중개 역할
   - Closeness Centrality: 네트워크 내 정보/자원 접근성

2. **특권성 지수 (Prestige Index)**
   - Bonacich의 권력 중심성 개념 활용
   - 영향력 있는 기업과의 연결이 더 높은 가중치를 가짐
   - 공급망 내 권력 구조 반영

3. **PageRank 알고리즘**
   - 본 프로젝트의 주요 분석 도구
   - 거래 관계의 방향성과 중요도를 동시에 고려
   - 기업의 영향력을 전역적(global) 관점에서 평가
   - 하청 구조 식별에 특히 효과적

### 2.2 가중치 접근 방식

우리는 여기에 네 가지 가중치 접근법을 추가로 도입하여 분석의 정교화를 시도합니다:
네트워크 분석에 사용된 가중치는 다음 네 가지 방식으로 구성했습니다:

1. **거래 강도 기반 (Transaction Intensity)**
   - 거래액과 거래기간 정보 활용
   - 가장 우수한 실증 결과 도출
   - 실제 거래 관계의 강도를 잘 반영

2. **관계 투자 기반 (Relationship Investment)**
   - 거래비용과 거래기간 활용
   - 장기적 거래 관계에 초점

3. **상호의존도 기반 (Interdependence)**
   - 거래액의 상대적 비중 고려
   - 기업 규모 대비 거래 중요도 반영

4. **복합 지표 기반 (Composite Index)**
   - 다양한 거래 특성의 통합적 고려
   - 요인분석을 통한 가중치 도출

이러한 가중치를 줌으로써 단순히 연결 관계만을 보기보다, 매출 등 시장 지배력을 나타내는 기업 특성과 기업간 거래의 특성을 이론적으로 더 잘 고려하고 분석 결과를 개선할 수 있음. 


## 3. 데이터 처리 및 분석

### 3.1 KOdata 거래데이터 전처리

원본 데이터가 가진 주요 문제점과 해결 방안:

1. **거래 방향 불일치**
   - 구매처/판매처 보고 정보의 비대칭성 존재
   - 구매처 보고 정보를 우선적으로 신뢰하여 처리

2. **기업 식별자 매칭**
   - KEDCD와 KEDCD_TRD 간 불일치 문제
   - 거래 관계 상호 매칭을 통한 데이터 보완

3. **결측치 처리**
   - 거래기간, 거래비용의 결측 패턴 분석
   - 가중치 계산 시 결측 정보 보정

### 3.2 R을 이용한 분석

분석은 크게 세 단계로 진행:

1. **비가중치 네트워크 분석**
   ```r
   # 기본 네트워크 구조 분석
   g_basic <- graph_from_data_frame(d = edges_df, directed = TRUE)
   pagerank_scores <- page_rank(g_basic)$vector
   ```

2. **방향성을 고려한 위계 분석**
   ```r
   # 구매-판매 방향성 분석
   in_degree <- degree(g_basic, mode = "in")
   out_degree <- degree(g_basic, mode = "out")
   ```

3. **가중치 기반 분석**
   ```r
   # 가중 네트워크 분석
   g_weighted <- graph_from_data_frame(d = edges_weighted, directed = TRUE)
   weighted_pagerank <- page_rank(g_weighted, weights = E(g_weighted)$weight)$vector
   ```
